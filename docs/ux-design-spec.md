## 1. 현황 분석

### 1.1 현재 페이지 구조

| 페이지 | 현재 역할 | 문제점 |
|--------|----------|--------|
| `/` (대시보드) | 내 조건 요약 + 추천 매물 + 최근 리포트 | 정보 우선순위 불명확, 매물 추가 후 변화 없음 |
| `/search` | 전체 매물 검색 | 관심 매물 상태 관리 부재 |
| `/property/[id]` | 매물 상세 + 뉴스/후기 링크 | 링크만 보여주는 건 의미 없음, 종합 리포트 필요 |
| `/reports` | 분석 리포트 이력 | 매물별 관계가 약함 |
| `/compare` | 매물 비교 | OK (유지) |
| `/settings` | 조건 설정 | OK (유지) |

### 1.2 현재 기술 스택

- **프론트엔드**: Next.js 16 (App Router), Tailwind CSS v4
- **아이콘**: lucide-react
- **상태관리**: TanStack React Query, nuqs
- **UX 라이팅**: 토스 스타일 (해요체, 짧은 문장, 쉬운 말)
- **디자인 토큰**: primary-500/600, neutral-50~900, rounded-xl/2xl

### 1.3 매물 상태 정의 (PropertyStatus)

```typescript
type PropertyStatus =
  | 'none'           // 미분류
  | 'interested'     // 관심
  | 'visit_planned'  // 방문 예정
  | 'visited'        // 방문 완료
  | 'candidate'      // 후보
  | 'rejected';      // 탈락
```

## 2. 핵심 문제점 및 해결 방향

### 2.1 문제 1: 대시보드 정보 구조 불명확

**현재**:
- "내 조건" 섹션이 너무 상세함 (6개 항목 카드)
- 추천 매물이 실제 관심 매물과 분리됨
- 분석 리포트가 하단에 묻힘

**개선**:
- 대시보드는 **진행 중인 액션** 중심으로 재구성
- 3개 영역: **진행 상황**, **오늘의 할 일**, **빠른 액션**

### 2.2 문제 2: 관심 매물 화면 부재

**현재**:
- 매물 상태(관심/방문예정/후보 등)를 설정할 수 있지만, 상태별로 모아보는 화면이 없음
- `/search`에서 필터로만 확인 가능

**개선**:
- `/favorites` 페이지 신규 추가
- 상태별 탭 네비게이션 (관심 / 방문예정 / 방문완료 / 후보 / 탈락)
- 칸반 보드 스타일 UI

### 2.3 문제 3: 매물 추가 위치 문제

**현재**:
- Header에 "매물 추가" 버튼 → 추가 후 대시보드에 변화 없음
- 사용자는 어디로 가야 할지 모름

**개선**:
- 매물 추가 후 자동으로 `/favorites`로 이동하고 "관심" 상태로 설정
- 토스트로 "관심 매물에 추가했어요" 안내

### 2.4 문제 4: 뉴스/후기 링크만 제공

**현재**:
- 상세 페이지에서 뉴스/후기 링크만 나열
- 사용자가 직접 읽어야 함 (비효율적)

**개선**:
- **AI 종합 리포트** 섹션 추가
- 뉴스 + 후기를 AI가 읽고 요약한 리포트 표시
- 기본 5개 참고, "더 많은 자료로 다시 분석" 버튼 제공

### 2.5 문제 5: 참고 데이터 확장 옵션 부재

**현재**:
- 분석 리포트는 고정된 데이터만 사용

**개선**:
- "기본 분석 (5개)" / "심층 분석 (20개)" 옵션 제공
- 심층 분석은 로딩 시간 안내 + 프로그레스바

## 3. 정보 구조 (IA) 재설계

### 3.1 네비게이션 구조

**Before (Header 기반)**:
```
홈(대시보드) | 검색 | 리포트 | 비교 | 설정
```

**After (탭 내비게이션 강화)**:
```
홈(대시보드) | 관심 매물 | 검색 | 비교 | 설정
```

**변경 이유**:
- "관심 매물"을 주요 탭으로 승격 (사용자의 핵심 작업 공간)
- "리포트"는 매물 상세 페이지 안에 통합

### 3.2 페이지별 목적 재정의

| 페이지 | 새로운 목적 | 핵심 CTA |
|--------|-----------|---------|
| `/` | 진행 상황 요약, 오늘 할 일 | "관심 매물 보기", "새 매물 찾기" |
| `/favorites` | 상태별 매물 관리 (칸반 보드) | "매물 추가", "상태 변경", "분석 요청" |
| `/search` | 전체 매물 검색 + 필터 | "관심 추가", "상세보기" |
| `/property/[id]` | 매물 상세 + AI 종합 리포트 | "리포트 생성", "관심 추가", "비교 추가" |
| `/compare` | 최대 3개 매물 비교 | 유지 |
| `/settings` | 조건 설정 | 유지 |

### 4.3 매물 상세 - `/property/[id]`

- 기존 5개 전문가 분석 유지 + 뉴스/후기 종합 추가 = 7개 섹션 통합 리포트
- 뉴스/후기 링크 나열 → AI가 읽고 요약한 리포트로 전환
- 별도 /reports 페이지 제거 → 상세 페이지에 통합

| 상황 | 문구 |
|------|------|
| 로딩 | "불러오고 있어요..." / "찾는 중..." |
| 빈 상태 | "아직 ~이/가 없어요" |
| 성공 | "~했어요" / "완료했어요" |
| 오류 | "~할 수 없었어요" / "다시 시도해 주세요" |
| 안내 | "~해 보세요" / "~할 수 있어요" |
| 버튼 | "~하기" / "~보기" (명사형 X) |

### 8.3 페이지별 주요 문구

**대시보드**:
- 헤더: "안녕하세요" / "오늘도 좋은 집을 찾아볼까요?"
- 진행 상황: "관심 매물 12개"
- 할 일: "방문 예정 매물 3곳 확인하기"
- 빈 상태: "아직 관심 매물이 없어요"

**관심 매물**:
- 헤더: "관심 매물" / "상태별로 매물을 관리하세요"
- 드래그 안내: "드래그해서 상태를 변경할 수 있어요"
- 빈 상태: "아직 관심 매물이 없어요"

**매물 상세 - AI 리포트**:
- 미생성: "아직 리포트가 없어요" / "뉴스와 후기를 분석해서 투자 의견을 정리해 드릴게요"
- 생성 중: "뉴스와 후기를 읽고 있어요" / "약 30초 소요돼요"
- 완료: "분석 완료 (2026년 2월 15일)"
- 버튼: "리포트 만들기" / "더 많은 자료로 다시 분석하기"

**토스트**:
- "관심 매물에 추가했어요"
- "상태를 변경했어요"
- "리포트 생성을 시작했어요"
- "조건이 저장됐어요"

## 9. 반응형 가이드

### 9.1 브레이크포인트

- **모바일**: `< 768px` (md)
- **데스크톱**: `>= 768px`

### 9.2 페이지별 반응형 전략

#### 대시보드
- **데스크톱**: 진행 상황 카드 가로 나열 (5개)
- **모바일**: 2×3 그리드 또는 세로 스크롤

#### 관심 매물
- **데스크톱**: 칸반 보드 (가로 스크롤 5개 열)
- **모바일**: 탭 네비게이션 + 리스트 뷰

#### AI 리포트
- **데스크톱/모바일**: 동일 (세로 스크롤)

## 10. 접근성 (a11y) 가이드

### 10.1 색상 대비

- 모든 텍스트는 WCAG AA 기준 (4.5:1 이상)
- 점수 뱃지는 색상 + 숫자 병기 (색맹 고려)

### 10.2 키보드 네비게이션

- 모든 인터랙티브 요소는 Tab으로 접근 가능
- 칸반 보드는 키보드로도 상태 변경 가능 (Shift+화살표)

### 10.3 스크린 리더

- 모든 버튼에 `aria-label` 제공
- 프로그레스바는 `role="progressbar"` + `aria-valuenow`

## 11. 예상 개선 효과

### 11.1 정량적 지표

- **매물 관리 효율**: 상태 변경 클릭 수 50% 감소 (칸반 드래그)
- **리포트 활용도**: AI 종합 리포트 읽기 비율 80% 이상
- **대시보드 체류 시간**: 20% 증가 (할 일 제안 효과)

### 11.2 정성적 개선

- **명확한 정보 구조**: 사용자가 "다음에 뭘 해야 할지" 바로 알 수 있음
- **관심 매물 중심 워크플로우**: 매물 추가 → 관심 → 방문 → 후보 → 계약
- **AI 리포트로 시간 절약**: 뉴스/후기 직접 읽기 불필요
