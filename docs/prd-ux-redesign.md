# PRD: 부동산 매물 분석 서비스 UX 개편

**작성일**: 2026-02-15
**버전**: v2.0
**목적**: 아파트 매매 리서치 프로세스를 중심으로 한 UX 전면 재설계

---

## 1. 서비스 목적 재정의

### 핵심 가치 제안
"아파트 매매를 위한 **리서치 과정**을 체계적으로 도와주는 서비스"

### 타겟 사용자가 겪는 문제
1. **정보 과부하**: 매물이 너무 많아서 어떤 것부터 봐야 할지 모름
2. **비체계적인 관리**: 본 매물, 관심 있는 매물, 방문 예정 매물 등이 섞여 있음
3. **깊이 있는 분석 부족**: 단순 스펙 비교로는 실제 살 만한 집인지 판단 어려움
4. **뉴스/후기 직접 찾기 귀찮음**: 링크만 모아놓은 것은 의미 없음

### 해결 방안
- **매물 라이프사이클 관리**: 발견 → 관심 → 분석 → 방문 → 결정까지 단계별 관리
- **AI 리포트 자동화**: 뉴스, 후기, 실거래가를 종합한 심층 분석
- **맞춤 대시보드**: 현재 내가 봐야 할 매물과 다음 액션을 명확히 제시

## 2. 사용자 저니 맵 (User Journey Map)

```
1단계: 조건 설정
   └→ 예산, 지역, 평수 등 내 조건 입력
   └→ 자동으로 조건에 맞는 매물 수집

2단계: 매물 발견 (Discovery)
   └→ 대시보드에서 "새로 나온 매물" 확인
   └→ 검색으로 특정 지역/단지 탐색
   └→ 마음에 드는 매물에 "관심" 표시

3단계: 분석 (Analysis)
   └→ 관심 매물 목록에서 매물 선택
   └→ 상세 페이지에서 "AI 분석 요청"
   └→ 뉴스/후기/시세를 종합한 리포트 확인
   └→ 리포트 기반으로 "방문 예정" 또는 "탈락" 결정

4단계: 방문 및 결정 (Decision)
   └→ 방문 예정 매물에 방문 날짜 메모
   └→ 방문 후 "방문 완료" 상태로 변경, 후기 메모
   └→ 최종 후보 2-3개 비교
   └→ "계약 진행" 상태로 변경

5단계: 관리 (Management)
   └→ 이력 조회 (언제 어떤 매물을 봤는지)
   └→ 탈락 매물 재검토
```

## 3. 페이지 구조 재설계

### 3.1 홈 (대시보드) - `/`

**목적**: 지금 내가 해야 할 일을 한눈에 보여줌

#### 섹션 구성
1. **내 조건 요약**
   - 예산, 지역, 평수 등 한눈에 확인
   - "조건 수정" 버튼

2. **진행 현황 카드**
   ```
   [관심 매물 5개] [방문 예정 2개] [방문 완료 3개] [후보 1개]
   ```
   - 각 카드 클릭 → 해당 상태 매물 목록으로 이동

3. **새로 나온 매물** (최근 7일 이내, 최대 6개)
   - 내 조건에 맞는 신규 매물
   - "관심 추가" 빠른 액션
   - "매물 더 찾기" 버튼

4. **분석 필요한 매물** (관심 상태이지만 아직 AI 분석 안 한 매물)
   - 최대 3개 표시
   - "AI 분석하기" 빠른 액션

5. **다음 방문 예정** (방문 예정 상태, 날짜순 정렬)
   - 방문 날짜, 주소, 메모
   - "길찾기" (카카오맵 연동)

#### 현재 대비 변경점
- ❌ 삭제: "추천 매물" 섹션 (너무 포괄적)
- ❌ 삭제: "최근 리포트" (리포트 페이지로 이동)
- ✅ 추가: "진행 현황 카드"
- ✅ 추가: "분석 필요한 매물"
- ✅ 추가: "다음 방문 예정"

### 3.2 매물 목록 (관심 매물) - `/properties`

**목적**: 상태별로 매물 관리

#### 탭 구조
```
[전체] [관심(5)] [방문예정(2)] [방문완료(3)] [후보(1)] [탈락(7)]
```

#### 각 탭 기능
- **전체**: 모든 매물 (미분류 제외)
- **관심**: 관심 표시한 매물, "AI 분석" 액션 버튼
- **방문 예정**: 방문 날짜 표시, "길찾기" 버튼
- **방문 완료**: 방문 날짜, 메모 표시
- **후보**: 최종 후보 매물, "비교하기" 버튼
- **탈락**: 탈락 이유 메모 표시, "재검토" 액션

#### 정렬/필터
- 정렬: 최신순, 가격순, 평수순, 분석 점수순
- 필터: 지역, 가격대, 평수 (내 조건 기반 미리 세팅)

#### 현재 대비 변경점
- 🆕 신규 페이지 (현재는 검색 페이지만 존재)
- favorites 테이블 → properties.status 기반으로 변경
- 빠른 상태 변경 액션 추가

### 3.3 매물 검색 - `/search`

**목적**: 새로운 매물 발견

#### 현재 유지
- 지도 기반 탐색 (추후 구현)
- 검색어 입력 (아파트명, 주소)
- 필터 (가격, 평수, 층수 등)

#### 개선사항
- 검색 결과에서 "관심 추가" 빠른 액션
- 검색 조건 저장 (자주 찾는 지역)
- "내 조건과 일치" 배지 표시

### 3.4 매물 상세 - `/property/[id]`

**목적**: 매물 상세 정보 + AI 분석 리포트

#### 섹션 구성

**1. 매물 헤더**
- 아파트명, 주소
- 가격, 평수, 층, 향, 연식
- 상태 변경 드롭다운 (관심 → 방문예정 → 방문완료 → 후보/탈락)
- "네이버 부동산 보기" 외부 링크

**2. AI 종합 분석 리포트** (핵심 변경)
```
┌─────────────────────────────────┐
│  종합 점수: 82점 (A)             │
│  [시세 85] [입지 90] [투자 75]  │
│  [규제 80] [리스크 78]          │
│                                  │
│  🔍 요약 (3줄)                   │
│  이 매물은 입지가 우수하나...     │
│                                  │
│  [상세 리포트 보기] 버튼         │
└─────────────────────────────────┘
```

**3. 뉴스/후기 종합 리포트** (✨ 새 기능)
```
❌ 이전: 뉴스 링크 5개 나열
✅ 신규: AI가 뉴스 5개 읽고 요약

[뉴스 종합 분석]
최근 3개월간 관련 뉴스 8건 분석 결과:
- ✅ 긍정: GTX 역세권 호재 (5건)
- ⚠️ 중립: 재건축 추진 논의 중 (2건)
- ❌ 부정: 학군 변경 가능성 (1건)

참고 기사: [기사1] [기사2] ...
[더 많은 기사로 재분석] 버튼
```

**4. 실거래가 차트**
- 최근 1년 거래 내역
- 평당 가격 추세

**5. 내 메모**
- 방문 날짜, 느낀 점, 체크리스트
- 사진 첨부 (추후)

#### 현재 대비 변경점
- ✅ 뉴스/후기 → AI 종합 분석으로 변경
- ✅ "더 많은 기사로 재분석" 버튼 추가
- ✅ 상태 관리 UI 개선 (칩 → 드롭다운)
- ❌ 별도 리포트 페이지 제거 (상세 페이지에 통합)

## 4. 매물 상태 (Status) 라이프사이클 정의

### 4.1 상태 정의

```typescript
type PropertyStatus =
  | 'none'           // 미분류 (수집만 됨, 아직 보지 않음)
  | 'interested'     // 관심 (눈여겨 보는 중)
  | 'visit_planned'  // 방문 예정 (날짜 예약됨)
  | 'visited'        // 방문 완료 (실제 가봤음)
  | 'candidate'      // 후보 (최종 후보 2-3개)
  | 'rejected';      // 탈락 (더 이상 고려 안 함)
```

### 4.2 상태 전환 플로우

```
수집 (none)
  ↓ [관심 추가]
관심 (interested)
  ↓ [AI 분석 → 괜찮다면]
  ↓ [방문 예약]
방문 예정 (visit_planned)
  ↓ [방문 완료]
방문 완료 (visited)
  ↓ [최종 후보 선정]
후보 (candidate)
  ↓ [비교 후 결정]
  └→ 계약 진행 (서비스 범위 밖)

※ 어느 단계에서든 [탈락] 가능
```

### 4.3 각 상태별 필수 액션

| 상태 | 필수 액션 | 선택 액션 |
|------|----------|----------|
| none | - | 관심 추가, 삭제 |
| interested | AI 분석 요청 | 방문 예약, 탈락 |
| visit_planned | 방문 날짜 입력 | 메모, 탈락 |
| visited | 방문 후기 메모 | 후보 등록, 탈락 |
| candidate | 비교하기 | 계약 메모 |
| rejected | 탈락 이유 메모 | 재검토 (interested로 복귀) |

## 5. AI 종합 리포트 시스템 재설계

### 5.1 핵심 원칙
AI 종합 리포트 = **기존 5개 전문가 분석** + **뉴스 종합 요약** + **임장 후기 종합 요약**
총 7개 섹션이 하나의 리포트로 구성되며, 매물 상세 페이지에서 직접 확인 가능.

### 5.2 리포트 구성 (7개 섹션)

| # | 섹션 | analysis_type | 역할 | 기존/신규 |
|---|------|--------------|------|----------|
| 1 | 시세 분석 | `market` | 호가 적정성, 주변 시세 비교, 가격 추이 | 기존 유지 |
| 2 | 입지 분석 | `location` | 교통, 학군, 편의시설, 생활환경 | 기존 유지 |
| 3 | 투자 분석 | `investment` | 예상 수익률, 미래 가치, 개발 호재 | 기존 유지 |
| 4 | 규제 분석 | `regulation` | 대출 한도, 세금, 규제지역 여부 | 기존 유지 |
| 5 | 리스크 분석 | `risk` | 건물 상태, 재해 위험, 안전 등급 | 기존 유지 |
| 6 | 뉴스 종합 | `news_summary` | 관련 뉴스 수집 → 긍정/중립/부정 분류 + 요약 | **신규** |
| 7 | 임장 후기 종합 | `review_summary` | 블로그 후기 수집 → 장단점 추출 + 요약 | **신규** |

### 5.3 각 섹션 출력 구조

#### 기존 5개 전문가 분석 (market, location, investment, regulation, risk)
기존 구조 유지:
```typescript
{
  score: number;          // 0~100
  grade: string;          // A+, A, B+, B, C, D
  summary: string;        // 2~3문장 요약
  strengths: string[];    // 강점
  weaknesses: string[];   // 약점
  recommendations: string[];
  details: Record<string, unknown>;
  confidence: number;
  data_sources: string[];
}
```

#### 신규: 뉴스 종합 (news_summary)
```typescript
{
  score: number;                    // 뉴스 긍정도 점수 (0~100)
  grade: string;
  summary: string;                  // 3~5줄 핵심 요약
  strengths: string[];              // 긍정 토픽들
  weaknesses: string[];             // 부정 토픽들
  details: {
    total_articles: number;
    positive_count: number;
    neutral_count: number;
    negative_count: number;
    key_topics: {
      topic: string;
      sentiment: 'positive' | 'neutral' | 'negative';
      article_count: number;
    }[];
  };
  data_sources: string[];           // 참고 기사 URL
  confidence: number;
}
```

#### 신규: 임장 후기 종합 (review_summary)
```typescript
{
  score: number;                    // 거주 만족도 점수 (0~100)
  grade: string;
  summary: string;                  // 3~5줄 핵심 요약
  strengths: string[];              // 장점 (조용함, 일조량 좋음...)
  weaknesses: string[];             // 단점 (주차 부족, 엘베 노후...)
  details: {
    total_reviews: number;
    frequently_mentioned: {
      keyword: string;
      count: number;
      sentiment: 'positive' | 'negative';
    }[];
  };
  data_sources: string[];           // 참고 후기 URL
  confidence: number;
}
```

### 5.5 분석 요청 플로우

#### 기본 분석
```
매물 상세 → "리포트 만들기" 클릭
  → Edge Function 호출
  → 1) 매물 정보 + 실거래 데이터 수집
  → 2) 뉴스 5~10건 크롤링 (네이버 뉴스 API)
  → 3) 임장 후기 5~10건 크롤링 (네이버 블로그 API)
  → 4) Claude API에 전체 데이터 전달
  → 5) 7개 분석 섹션 순차 생성 (기존 5개 + 뉴스 + 후기)
  → 6) analysis_reports 테이블에 7건 INSERT
  → Realtime으로 진행 상황 표시 (0/7 → 7/7)
```

#### 심화 분석 (뉴스/후기 확장)
```
기본 리포트 확인 → "더 많은 자료로 다시 분석하기" 클릭
  → 뉴스 20건 + 후기 20건 추가 수집
  → news_summary, review_summary 2개 섹션만 재분석
  → 기존 5개 전문가 분석은 유지
  → 업데이트된 리포트 표시
```

### 5.6 DB 변경사항

## 11. 성공 지표 (KPI)

### 사용자 행동 지표
- **활성 매물 수**: 사용자당 평균 관심 매물 개수 (목표: 5개 이상)
- **분석 요청률**: 관심 매물 중 AI 분석 요청 비율 (목표: 70%)
- **방문 전환율**: 관심 매물 → 방문 예정 전환율 (목표: 30%)
- **후보 등록률**: 방문 완료 → 후보 전환율 (목표: 50%)

### 리포트 품질 지표
- **재분석 요청률**: 기본 분석 후 심화 분석 요청 비율 (목표: 20%)
- **리포트 유용성**: 사용자 피드백 (5점 만점, 목표: 4점 이상)

### 서비스 이탈 방지
- **7일 리텐션**: 가입 후 7일 내 재방문율 (목표: 60%)
- **평균 매물 관리 기간**: 첫 관심 등록 ~ 계약/탈락까지 (추적용)
